# DM-life 四项修复总览（2026-07-16）

## 1. 运行模式独立标签 ✅
**文件**：`packages/web-collab/src/components/SettingsPage.tsx`
- `CATEGORIES` 首项新增 `{ id: 'mode', label: '运行模式' }`，位于「数据」标签**上方**。
- 新增 `category === 'mode'` 独立分支渲染运行模式开关；「数据」标签内不再内嵌运行模式块。
- 验证：`tsc --noEmit` EXIT 0、`vite build` EXIT 0（1800 模块）。

## 2. 添加任务后无反应 ⚠️ 根因=旧孤儿引擎进程（非当前代码缺陷）
- 起隔离引擎实例（PORT=14590、独立数据目录）端到端复现：用 web-collab 与 packages/web 两种真实 payload 跑 `tasks.create` → `tasks.today`，**任务均成功创建并出现在今日视图**。
- 当前代码的写路径（CommandPalette `create.mutate` → `onSuccess` 里 `utils.tasks.today.invalidate()` → 看板 `useQuery` 刷新）完全正确。
- 用户侧「无反应/无报错」是项目已知的**端口孤儿进程**问题（旧 engine 占 14570、加载旧代码）。
- **修复动作**：干净重启引擎即可。用 `start-dm-life.bat`（会自动杀孤儿进程+协商端口），或手动停掉旧 engine 后重启。重启后访问 `http://127.0.0.1:<实际端口>/_routes` 可确认运行中引擎的 procedures 列表含 `tasks.create`。
- 未改动 create/显示代码，避免引入新 bug。

## 3. 灵感和记事无法保存（notes embedding 列缺失）✅
**文件**：`packages/engine/src/db/migrate.ts`
- 根因：旧库 `CREATE TABLE IF NOT EXISTS` 不补列，保存 notes 报 `table notes has no column named embedding`。
- 修复：增量补列 `notes.embedding` / `embedded_at`（幂等 `addColumnIfMissing`，列已存在则忽略）。
- 验证：engine `vitest run` **52/52** 通过。

## 4. 协作页面共享事项显示不全 → 选「方案 B：导入个人页」✅
**新增**：`packages/web-collab/src/features/shared/importToLocal.ts`
- `isImportable(item)` + `importSharedToLocal(utils, item)`，复用已验证的本地 mutation（`notes.ingest` / `tasks.create` / `reminders.create` / `interests.capture`），**零新增 API**。
- 关键修正：`useUtils()` 对 mutation 仅暴露 `setMutationDefaults/getMutationDefaults/isMutating`，**无 `.mutate()`**；正确写法是 `utils.client.X.mutate(...)`。类型用 `ReturnType<typeof trpcLocal.useUtils>`。

**改动**：`packages/web-collab/src/features/shared/FamilySharedItemsBoard.tsx`
- 在卡片 `renderSnapshot` 下方渲染「导入我的页面」按钮（`isImportable` 时显示），点击经 `trpcLocal.useUtils()` 的 client 发起写、成功后 `localUtils.invalidate()` 刷新个人页并 toast 成功/失败。
- 验证：`tsc` + `vite build` 均 EXIT 0。
