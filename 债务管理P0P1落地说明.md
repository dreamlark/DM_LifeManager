# 债务管理银行评估 P0/P1 落地说明

> 本文档对应《债务管理银行专业评估.md》中的 P0/P1 级建议项，说明各项的**评估逻辑、数据来源与展示格式、交互流程**。
> 技术栈：TS 引擎（Drizzle + sql.js + 纯函数计算） + React 前端（tRPC 查询/变更）。所有计算在引擎侧完成，前端仅做展示与采集。

## 一、P0-1 利率重定价自动化（LPR / 基准利率联动）

### 评估逻辑
原设计需用户逐期手填「利率重定价记录」，易错且不可持续。银行实践是：**合同锁定向加点，执行利率随挂钩基准（LPR_1Y / LPR_5Y / 央行基准）按约定周期自动重算**。

### 数据模型
- 新增后端字段 `repricing`（JSON）：`{ benchmark, spread, cycleMonths, anchor, fixedDate? }`
  - `benchmark`：LPR_1Y / LPR_5Y / PBOC_BASE
  - `spread`：签约时锁定的加点（百分点，永久不变）
  - `cycleMonths`：重定价周期（12=年 / 6=半年 / 3=季）
  - `anchor`：anniversary（对年对月对日）/ fixed_date（固定日历日）
- 引擎 `generateRepricingAdjustments(rule, startDate, termMonths)` 据此自动派生 `(生效日, 新利率)` 列表，生效日取「当日基准 + 锁定加点」。
- 内置 `LPR_HISTORY`（2022–2026 各品种利率），无法联网时作为基准，可随央行公布更新。
- **优先级**：`repricing` 规则存在时**自动覆盖**手填 `rateAdjustments`（手填列表保留为手动覆盖通道）。

### 前端交互
- 债务弹窗（新增/编辑）中，利率类型选 `LPR` 或 `基准利率` 时出现「启用自动重定价」勾选。
- 勾选后展示：基准品种 / 加点 / 重定价周期 / 对日方式；选「固定日历日」再补一个日期。
- 固定利率下不出现该区块（重定价无意义）。

### 展示格式
- 还款清单中每期 `rate` 列即为当期执行利率；重定价生效月利率自动跳变（如 4.9% → 3.9%）。

## 二、P0-2 实际已还勾稽（计划 vs 真实流水）

### 评估逻辑（评估报告中标记为「最严重缺口」）
原先进度条只反映**计划时间口径**（已扣期数 / 总期数），与「真实已还本金」两张皮。银行对账要求：**计划还款额 = 实际入账流水**。

### 评估逻辑 / 数据来源
- 后端 `debtProgressSummary` 同时返回：
  - `paidPrincipal`：按计划（时间口径）应已还本金
  - `actualPaidPrincipal`：按 `debt_payment` 真实流水条数，取还款计划前 N 期本金累计（N = 真实还款笔数）
- 差异 `delta = actualPaidPrincipal − paidPrincipal`：
  - `> 0.5`：提前还/多还（绿色，省利息）
  - `< -0.5`：存在逾期或漏还（琥珀色，风险提示）
  - 接近 0：账实相符

### 前端展示
- 进度浮层（右上角标签）整体区新增「实际已还」一行。
- 单笔债务展开「还款清单」顶部新增**实际已还勾稽卡**：计划已还 / 实际已还 / 差异。
- 单笔进度浮层内每条债务下方显示「实际已还 ¥X」。

## 三、P1-1 实际年化利率（IRR / EAR）披露

### 评估逻辑
名义 APR 不等于真实资金成本。对借款人，**IRR = 内部收益率（月供现金流折现）**，年化即 EAR = (1+月IRR)^12 − 1；等额本息下 EAR 高于名义 APR（反映资金被按月占用）。含提前还款时 IRR 更贴近真实成本。

### 数据来源
- 引擎 `getDebtSummary` 新增 `irr` 字段（借款人视角现金流：t0 收本金(+)，其后每月付月供+提前还款(−)，二分法求解，再年化）。

### 前端展示
- 债务还款清单统计区新增「实际年化(IRR)」项（天蓝色高亮）。

## 四、P1-2 提前还款收益量化

### 评估逻辑
提前还款的价值不应只显示「已还」，应量化**节省利息 + 缩短期限**，辅助决策是否提前还。

### 数据来源
- 引擎对比「无提前还款基准计划」与「实际计划」：
  - `interestSaved` = 基准总利息 − 实际总利息
  - `termShortened` = 基准期数 − 实际期数

### 前端展示
- 仅当 `interestSaved > 0` 时，还款清单内显示绿色**提前还款收益卡**：节省利息 / 缩短 N 期。

## 五、P1-3 最优还款策略建议（雪崩 / 滚雪球）

### 评估逻辑
- **雪崩法（avalanche）**：利率降序，先还利率最高的债 → 总利息最小（银行最优）。
- **滚雪球法（snowball）**：余额升序，先还清最小额 → 心理正反馈强。
- 手动 `priority` 字段存在时**置顶**（同档内仍按策略排序），支持用户自定义优先级。

### 数据来源
- 后端 `debtPayoffAdvice(mode)`：基于引擎摘要（当前利率、剩余本金、月供）排序，返回 `{ rank, creditor, currentRate, remainingPrincipal, reason }`。

### 前端交互
- 进度浮层内「最优还款策略」区块，提供 `雪崩法 / 滚雪球` 切换按钮（即时切换，无需保存）。
- 列表显示排名、债务名、理由（如「利率最高 4.9%，先还省利息」）。
