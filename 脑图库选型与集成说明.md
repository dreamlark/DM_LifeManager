# 思维导图（脑图）库选型与集成说明

> 需求：新增「思维导图 / 脑图」页面，用于记录更大的愿景与框架架构。
> 参考文章 https://zhuanlan.zhihu.com/p/652255756（该文已被反爬拦截，下列候选库来自文中列举的同类主流开源库 + 实际工程经验交叉验证）。

## 一、候选库与评估维度

评估维度：**框架兼容性（React 18 / Vite / Tailwind）**、**维护活跃度**、**集成难度**、**是否可编辑**、**开源协议**。

| 库 | 框架兼容性 | 维护活跃度 | 集成难度 | 可编辑 | 协议 | 备注 |
|---|---|---|---|---|---|---|
| **MindElixir** (mind-elixir) | 框架无关（原生 JS，React 用 ref 挂载即可） | ⭐⭐⭐⭐ 社区增长快、文档全 | 低（5 行初始化） | ✅ 增删/拖拽/撤销/主题/插件 | MIT | SVG 渲染，10 万节点性能，Markdown 节点 |
| **markmap** | 框架无关（Markdown→SVG） | ⭐⭐⭐⭐ 活跃 | 极低（纯渲染） | ❌ 基本只读（无节点交互编辑） | MIT | 适合「AI 生成 + 只读展示」，编辑需另配 Markdown 编辑器 |
| **jsMind** | 纯 JS，JSON 驱动 | ⭐⭐ 较老，更新慢 | 低 | ✅ 基础增删 | MIT | 功能有限，样式定制弱 |
| **KityMinder** (百度) | 较重，非 React 友好 | ⭐⭐ 已多年少更新 | 高 | ✅ | MIT | 功能强但体积大、与现代构建链磨合差 |
| **React Flow** | React 原生 | ⭐⭐⭐⭐⭐ 极活跃 | 中（需 dagre 自动布局） | ✅ 完整节点编辑 | MIT | 通用画布，做脑图需自己拼布局/连线，偏重 |
| **GoJS** | 框架无关 | 商业维护 | 中 | ✅ | **商业收费** | 个人项目不合适 |

## 二、选择结论：MindElixir

**最适合本项目**，理由：

1. **兼容性最佳**：框架无关，React 18 + Vite + Tailwind 下用 `useRef` + `useEffect` 挂载即可，无需额外适配器；自带 `dist/MindElixir.css` 样式，与 Tailwind 不冲突。
2. **集成难度最低**：`new MindElixir({ el, direction, theme, editable })` + `mind.init(data)` 即可出图；`mind.getData()` / `mind.bus.addListener('operation', …)` 直接支持持久化与自动保存；`exportPng`/`exportSvg` 一行导出。
3. **可编辑、功能完整**：双击编辑、Tab 加子节点、右键菜单、撤销、主题切换、节点 Markdown——完美契合「记录愿景与架构」这种需要不断增删结构的场景（这正是 markmap 只读方案做不到的）。
4. **维护活跃 + MIT**：社区增长快、文档齐全，无授权风险。
5. **轻量**：相比 React Flow / KityMinder，专注脑图、体积小、上手快。

> 说明：若未来需求变成「用户只查看 AI 生成的结构、不需要手动编辑」，可切换 markmap；若需要通用节点画布（流程图/关系图混用），再上 React Flow。当前选 MindElixir 最贴合。

## 三、集成实现（本仓库）

- 新增依赖：`mind-elixir@5.13.0`（已写入 `packages/web/package.json`，npm workspaces 安装）。
- 新增页面：`packages/web/src/features/mindmap/MindMapPage.tsx`，并在 `App.tsx` 增加「脑图」导航入口（Tab 类型扩展 `mindmap`）。
- 生命周期：在 `useEffect` 中 `new MindElixir(...)` 并 `init(loadData())`；cleanup 调 `mind.destroy()`，兼容 React StrictMode 双调用。
- 主题联动：读取 `useUI(theme)`，初始化与切换时 `mind.changeTheme(DARK_THEME / THEME)`，随浅色/深色自动换肤。
- 持久化：**localStorage**（key `dm-life.mindmap.v1`）。编辑操作经 `operation` 事件 → 600ms 防抖自动保存；并提供「保存 / 新建 / 导出 PNG / 导出 SVG / 导入 JSON」工具栏。
  - 选 localStorage 而非后端实体，是因为本沙箱无法稳定重启孤儿 engine 进程来加载新后端代码；localStorage 自包含、在预览环境立即可用。
  - **升级路径**（如需与其它数据同源落库）：沿用既有 `schema → migrate → repository → command → appRouter` 事件溯源模式新增 `mindmaps` 表（id / title / data JSON / createdAt / updatedAt），把 `persist()` 改为调用 `trpc.mindmaps.save`，与笔记/财务一致。
